# Перечень используемых инструментов с обоснованием выбора
* Android Studio - среда разработки в которой будут писаться тесты.
* Java - язык программирования на котором будут написаны тесты.
* Gradle - система автоматической сборки, для создания и сборки проекта.
* JUnit 4 - фреймворк для создания автотестов.
* SDK - набор средств разработки для работы с эмуляторами/физическими девайсами.
* Git - место для хранения и обновления тестов
* Samsung A50 - мобильник для прогонки тестов
* эмулятор Nexus 6P API 30 - эмулятор мобильника
* Espresso - фреймворк для написания тестов
* Allure - фреймворк для сбора статистики прогонов тестов


# Перечень и описание возможных рисков при автоматизации
* Наличие соединения с интернетом
* Не все может быть автоматизировано при наличии критических ошибок


# Интервальная оценка с учётом рисков (в часах)
* 80 часов (2 недели, рабочие часы)


#План сдачи работ (когда будут автотесты, результаты их прогона и отчёт по автоматизации)
* создание тест-плана
* создание автотестов
* прогон тестов и создание отчета по результатам, возможно с мелкими правками тестов
